 Aputuogya D/A JHS Website

A modern, responsive school website built to showcase Aputuogya D/A JHS, featuring a clean interface, image gallery, and a Supabase-powered contact system.

---

 Overview

This project is a single-page school website designed to:

- Present information about the school
- Display a gallery of school activities
- Allow users to send messages via a contact form
- Demonstrate a simple full-stack setup using Supabase

---

 Features

-  Home Section – Introduction and branding
-  About Section – School focus and mission
-  Gallery – Displays images hosted on Supabase Storage
-  Contact Form – Sends messages to Supabase database
-  UI Design
  - Dark theme
  - Blurred background effect
  - Classic serif typography

---

 Tech Stack

- HTML5
- CSS3
- JavaScript (ES Modules)
- Supabase
  - Database (PostgreSQL)
  - Storage (Images)

 Setup Guide

1. Clone the Repository

git clone https://Simms-creator.github.io/ajs/
cd ajs

---

2. Connect Supabase

Go to your Supabase dashboard and copy:

- Project URL
- Anon Public Key

Update your code:

const supabase = createClient(
  "https://pcfgbsoetouzasxpovqo.supabase.co",
  "sb_publishable_XC27I6E9iq7OcDsMXP5EGQ_5tvCsy4O"
);

---

3. Create the Database Table

Run this SQL in Supabase:

create table messages (
  id bigint generated by default as identity primary key,
  name text not null,
  email text not null,
  message text not null,
  created_at timestamptz default now()
);

---

4. Enable Row-Level Security (RLS)

- Go to Table Editor → messages
- Enable RLS

---

5. Add Insert Policy

Run:

create policy "Allow public insert"
on messages
for insert
with check (true);

---

6. Run the Project

Simply open:

index.html

in your browser.

 Image Handling

Images are stored in Supabase Storage and used via public URLs:

https://PROJECT_ID.supabase.co/storage/v1/object/public/BUCKET/FILE

Make sure:

- Bucket is set to public
- Files exist and are correctly named

---

 Security Notes

- Use only the anon public key in frontend apps
- Never expose the service role key
- Enable RLS for database protection
